# Pesquisa - Lacos de Repeticao 
## 1. O que e uma estrutura de repeticao? 
Resposta: É um recurso na programação usado para executar um bloco de código várias vezes seguidas automaticamente. Em vez de você escrever a mesma instrução dez vezes, você usa uma estrutura de repetição para dizer ao programa que faça aquilo dez vezes, poupando linhas de código.
## 2. Para que serve o for? 
Resposta: O for serve para quando você já sabe antecipadamente o número exato de vezes que o código precisa ser repetido. Ele é ideal para contagens, pois permite definir onde a repetição começa, qual é a condição para ela parar e como ela avança (de 1 em 1, por exemplo), tudo em uma única linha.
## 3. Para que serve o while? 
Resposta: O while (enquanto) serve para quando você não sabe quantas vezes o código vai precisar se repetir, pois a parada depende de uma condição que pode mudar a qualquer momento. Ele testa essa condição antes de rodar o código. Se a condição for falsa logo de início, o código dentro dele não é executado nenhuma vez.
## 4. Para que serve o do-while? 
Resposta: O do-while (faça-enquanto) serve para os casos onde você precisa que o código seja executado pelo menos uma vez antes que qualquer teste seja feito. Ao contrário do while comum, ele primeiro roda o bloco de código e só depois checa se deve continuar repetindo ou não.
## 5. Diferenca entre os tres lacos 
Resposta:A diferença está em como eles controlam o momento da repetição:

O for é focado em quantidade: você usa quando sabe o número de voltas que o programa deve dar. Ele testa a condição no início.

O while é focado em uma condição: você usa quando não sabe o número de voltas, e ele também testa a condição no início (podendo rodar zero vezes).

O do-while é focado na garantia de execução: você usa quando o código precisa rodar obrigatoriamente a primeira vez, já que o teste da condição só é feito no final de cada volta.
